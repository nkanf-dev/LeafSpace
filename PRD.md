## 1. 产品定位

### 1.1 一句话定义

**LeafSpace 是面向扫描版 PDF 的深度阅读工作区，让用户像翻纸书一样快速定位，并像在桌面上摊开资料一样并行对比。**

### 1.2 产品命名

- 新产品名：**LeafSpace**
- 历史名称：**PaperBook Reader**
- 本文档以后统一使用 **LeafSpace**

### 1.3 核心问题

现有 PDF 阅读器主要解决“看一页”的问题，但不能很好解决以下问题：

1. **快速定位**：扫描版 PDF 缺乏文本搜索优势，用户只能滚动、试错、来回跳。
2. **多页对比**：研究、复习、法律审阅往往需要同时看 2~5 页内容。
3. **空间记忆**：用户记得“在书的前半部分”“靠后几页”，但普通阅读器缺少这种位置感。
4. **工作区连续性**：用户今天搭好的参考页组合，明天无法自然恢复。

### 1.4 产品价值主张

LeafSpace 用四个核心能力还原纸质书体验并增强数字能力：

- **Quick Flip / 速翻**：像快速翻纸书一样浏览整本内容。
- **Page Holding / 夹页**：像手指夹住页面一样保留关键页。
- **Timeline / 时间轴定位**：建立整本书的位置感。
- **Workspace / 工作区**：把夹页展开为并行窗口，形成对比阅读空间。

## 2. 目标用户与场景

### 2.1 目标用户

#### P0 核心用户
- 学术研究者：对比论文引言、图表、方法、结论
- 法律从业者：对照条款、案例、附录
- 备考学生：快速复习教材、标记重点页、反复对照
- 技术/产品人员：阅读扫描版规范、手册、档案

#### P1 延伸用户
- 漫画/图录读者
- 历史文档、古籍、馆藏资料阅读者

### 2.2 典型场景

#### 场景 A：论文对比阅读
1. 用户阅读第 12 页引言
2. 想对比第 46 页实验图表、第 79 页方法、第 121 页结论
3. 通过速翻快速定位
4. 将关键页夹入面板
5. 打开多个对比窗并保存工作区

**成功标准**：用户可在 30 秒内建立 3 页以上的对比工作区。

#### 场景 B：考试复习
1. 用户只记得重点在“前半本”和“最后几章”
2. 进入速翻模式快速浏览
3. 连续夹住 8~12 个重点页
4. 回到工作区逐个复习

**成功标准**：2 分钟内完成 10 页重点标记，并可快速切换复习。

#### 场景 C：法律/合同核对
1. 用户需要核对主条款、定义、附表、例外条款
2. 将 4 个页面分别固定到工作区
3. 在主窗与对比窗之间反复切换

**成功标准**：无需反复跳页即可持续完成对照工作。

## 3. 产品原则

1. **先支持扫描版 PDF 的核心阅读闭环，再做增强功能**
2. **先让“定位、夹页、对比、恢复工作区”顺畅，再追求炫技动画**
3. **桌面端优先，移动端做不破坏核心模型的轻量适配**
4. **以空间记忆和并行阅读效率为第一目标**
5. **v1.0 控制范围，不引入 OCR、AI、协作等重模块**

---

## 4. 版本目标

### 4.1 v1.0 目标

交付一个可稳定使用的 **扫描版 PDF 深度阅读工作区**，具备：

- PDF 基础阅读
- 速翻浏览
- 夹页管理
- Timeline 定位
- 多窗口/分屏对比
- 工作区保存/恢复

### 4.2 北极星指标

- 夹页功能使用率 > 70%
- 平均单次会话打开对比窗口数 > 2
- 工作区保存使用率 > 30%
- 7 日留存率 > 40%

### 4.3 可用性指标

- 首屏可阅读时间 < 3 秒（常规文档）
- 常规交互稳定 60fps
- 支持 100MB 级扫描版 PDF 的可用阅读
- 工作区恢复成功率 > 95%

## 5. 功能范围

### 5.1 P0（v1.0 必做）

#### 5.1.1 基础阅读
- 使用 PDF.js 渲染扫描版 PDF
- 单页阅读为默认模式
- 缩放范围：50% - 400%
- 上一页 / 下一页导航
- 当前页前后若干页预加载

#### 5.1.2 缩略图系统
- 后台生成缩略图
- 统一中等分辨率为默认
- 支持 LRU 缓存
- 被夹页面必须优先有缩略图

#### 5.1.3 速翻模式（Quick Flip）
- `Space` 进入/退出速翻
- 横向胶片式预览带
- 当前页高亮
- `←` / `→` 导航
- 支持短按精准、长按加速
- 速翻中可直接夹页、跳转、在新窗打开

#### 5.1.4 夹页系统（Page Holding）
- 当前页加入夹页列表
- 从夹页列表取消
- 夹页显示缩略图、页码、默认标题
- 支持排序
- 支持有限数量上限（默认 12）

#### 5.1.5 Timeline
- 显示当前阅读位置
- 显示夹页标记点
- 显示目录/章节标记（有目录时）
- 点击跳转，拖动预览

#### 5.1.6 工作区 / 窗口系统
- 主窗口始终存在
- 支持对比窗口
- 支持浮动窗口（受控上限，默认最多 4 个）
- 支持基础吸附布局：
  - 左右分屏
  - 双列对比
  - 简单平铺预设
- 单击夹页可打开页面
- Shift / 快捷键支持“在新窗打开”
- 可将某个窗口设为主窗

#### 5.1.7 工作区保存/恢复
- 保存打开的页面、窗口布局、活动页、夹页列表
- 重新打开文档后恢复最近工作区

### 5.2 P1（v1.1 优先增强）
- 双页展开阅读
- 更完整的窗口吸附与布局预设
- 更细粒度的缩略图尺寸系统（S/M/L/A）
- 夹页备注、颜色标签、自定义名称
- 移动端 bottom sheet 式工作区
- 更丰富的快捷键与窗口管理器视图

### 5.3 P2（后续版本）
- 3D 翻页效果（作为可切换阅读皮肤/模式）
- OCR / 文本搜索 / 重排
- 云同步
- 协作与分享
- AI 辅助总结、问答、定位
- 摄像头手势 / AR 交互

## 6. 明确不做（v1.0）

以下内容不进入 v1.0 主交付：

- ❌ OCR 与全文搜索
- ❌ AI 摘要、AI 问答、智能推荐
- ❌ 云同步与多人协作
- ❌ 批注/高亮体系
- ❌ 复杂自动排版与文本重流
- ❌ 摄像头手势 / AR 功能
- ❌ 强依赖 3D 翻页的首版主流程

> 说明：3D 翻页可保留技术扩展位，但不作为 v1.0 上线门槛。

## 7. 详细功能规格

### 7.1 基础阅读

#### 桌面端
- `←` / `→`：翻页
- 滚轮 / 快捷键：缩放
- `Home` / `End`：首页/末页

#### 移动端（轻量适配）
- 左右滑翻页
- 双指缩放
- 长按缩略图呼出操作

### 7.2 速翻模式

#### 进入 / 退出
- 桌面端：`Space`
- 退出：`Esc` 或再次 `Space`

#### 操作
- `←` / `→`：移动选中页
- 长按加速：
  - <200ms：1 页
  - 200~500ms：3 页
  - 500~800ms：10 页
  - >800ms：进入连续加速
- `Enter`：跳转到该页
- `↑`：夹页
- `N`：在新窗打开
- `Shift+Enter`：对比打开
- `J`：仅跳转并退出速翻

### 7.3 夹页系统

#### 数据结构（产品层）
每个夹页至少包含：
- 页码
- 缩略图
- 默认名称
- 创建时间
- 是否已在窗口中打开

#### 交互
- 单击：在当前焦点窗打开
- 双击：主窗对比打开
- Shift+单击：新建窗口打开
- 删除夹页时，如有关联窗口，需提示是否同步关闭

### 7.4 Timeline
- 当前位置滑块
- 夹页点
- 目录点（可选）
- 拖动时显示实时页码预览
- 点击后直接跳转

### 7.5 窗口系统

#### 窗口类型
- 主窗（Main）
- 对比窗 / 浮动窗（Floating）
- 吸附窗（Docked）

#### v1.0 限制
- 默认最多同时显示 5 个阅读窗口（含主窗）
- 吸附布局只做常用预设，不做复杂自由布局算法
- 移动端不提供完整桌面式拖拽窗口，改为面板/卡片式浏览

### 7.6 工作区
- 每个 PDF 独立保存最近工作区
- 内容包括：窗口布局、窗口页码、夹页状态、当前页、缩放、面板偏好

## 8. 关键交互与快捷键

### 8.1 桌面端快捷键
| 快捷键 | 功能 |
|--------|------|
| `Space` | 进入/退出速翻 |
| `Esc` | 退出速翻 / 关闭当前临时层 |
| `←` `→` | 翻页 / 速翻中移动 |
| `↑` | 夹页 |
| `↓` | 取消夹页 |
| `Enter` | 打开当前选中页 |
| `Shift+Enter` | 对比打开 |
| `N` | 在新窗打开 |
| `J` | 仅跳转 |
| `Ctrl/Cmd + Tab` | 切换活动窗口 |
| `Ctrl/Cmd + W` | 关闭当前非主窗 |
| `Home` / `End` | 首页 / 末页 |

### 8.2 移动端原则
- 不复制桌面端全部复杂窗口能力
- 核心保留：阅读、速翻、夹页、查看已夹页、切换对比页
- 工作区通过 bottom sheet / 卡片列表表达

## 9. 非功能要求

### 9.1 性能
- UI 主流程操作不卡顿
- 缩略图后台生成，不阻塞阅读
- 大 PDF 采用惰性加载与缓存淘汰

### 9.2 稳定性
- 文档加载失败可重试
- 缩略图生成失败不阻塞主阅读
- 工作区恢复失败时可降级回单窗

### 9.3 可扩展性
- 后续可接入 OCR、AI、云同步
- 后续可升级 3D 翻页模式

## 11. 最终判断

LeafSpace v1.0 不是“另一个 PDF 阅读器”，而是一个**面向扫描版 PDF 的空间化深度阅读工作区**。

首版的核心不是做尽可能多的功能，而是把下面这条闭环做扎实：

**打开 PDF → 快速定位 → 夹住关键页 → 展开对比 → 保存工作区 → 下次继续阅读**
